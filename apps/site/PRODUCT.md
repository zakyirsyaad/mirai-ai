# Product

## Register

brand

## Users

Mirai's public website is for CROO Marketplace buyers, AI-agent builders, and MCP-client users who want to run X campaigns from tools such as Claude Code, Codex, Cursor, Hermes, or another MCP client. They arrive from a marketplace listing, a docs link, or a demo and need to understand what Mirai does, how access works, and how to install it without creating a dashboard account.

## Product Purpose

The site explains Mirai as an MCP-first local-client experience backed by a hosted worker runtime. Success means a buyer can trust the CROO purchase flow, understand the two services, install `@mirai-agent/mcp`, generate the right client config, and know that posting, expiry, and proof-of-work are handled by Mirai after license activation.

## Brand Personality

Precise, capable, and slightly futuristic. Mirai should feel like infrastructure for serious agent work, with enough motion and interface detail to make the product feel alive without becoming a generic SaaS landing page.

## Anti-references

Avoid dashboard-first SaaS pages, vague AI automation claims, decorative card grids with no workflow substance, fake analytics screenshots, and anything that implies users must manage local Docker, Redis, Postgres, or a long-running worker.

## Design Principles

- Show the agent surface, not just marketing copy.
- Make MCP setup feel concrete and copyable.
- Keep trust close to the workflow: signed licenses, hosted worker, expiry checks, and encrypted X access.
- Treat CROO as the purchase system and Mirai as the execution system.
- Stay concise; the buyer should reach install instructions quickly.

## Accessibility & Inclusion

Target WCAG AA contrast for all core text and controls. Keep motion restrained, respect reduced-motion preferences, and make install/config actions usable with keyboard navigation and clear button labels.
